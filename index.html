npm ဆိုတာ ဘာလဲ?

Node Package Manager အကြောင်း အသေးစိတ်လေ့လာခြင်း

ပြီးပြည့်စုံသော လမ်းညွှန် (Slide ၃၀)

npm ရဲ့ အရှည်ကောက်

npm ဆိုတာ Node Package Manager ရဲ့ အတိုကောက်ဖြစ်ပါတယ်။

  • Node.js အတွက် အဓိက သုံးတဲ့ Package Manager တစ်ခုပါ။
  • JavaScript Code တွေကို အလွယ်တကူ မျှဝေနိုင်၊ ပြန်လည်အသုံးပြုနိုင်အောင် ဖန်တီးထားတာဖြစ်ပါတယ်။
  • ကမ္ဘာပေါ်မှာ အကြီးမားဆုံး Software Registry (ဒေတာဘေ့စ်ကြီး) တစ်ခုလည်း ဖြစ်ပါတယ်။

npm ရဲ့ သမိုင်းကြောင်း

  • ၂၀၁၀ ခုနှစ်မှာ Isaac Z. Schlueter ဆိုသူက စတင်ဖန်တီးခဲ့ပါတယ်။
  • Node.js developer တွေ သူတို့ရေးထားတဲ့ code တွေကို အလွယ်တကူ share နိုင်ဖို့ ရည်ရွယ်ခဲ့တာပါ။
  • ယနေ့ခေတ်မှာတော့ Front-end (React, Vue, Angular) နဲ့ Back-end (Node.js) အားလုံးအတွက် မရှိမဖြစ် လိုအပ်တဲ့ Tools တစ်ခုဖြစ်လာပါပြီ။

ဘာကြောင့် npm ကို သုံးတာလဲ?

Code တွေကို အစကနေ ပြန်ရေးနေစရာ မလိုတော့ပါဘူး။

  • တခြား Developer တွေ ရေးသားထားပြီးသား Code (Package) တွေကို ကိုယ့် Project ထဲ အလွယ်တကူ ထည့်သွင်းအသုံးပြုနိုင်ဖို့။
  • Project တစ်ခုမှာ ပါဝင်တဲ့ Code တွေရဲ့ Version တွေကို စနစ်တကျ စီမံခန့်ခွဲနိုင်ဖို့။
  • Project ကို လွယ်ကူစွာ Update လုပ်နိုင်ဖို့နဲ့ Share နိုင်ဖို့ ဖြစ်ပါတယ်။

Package ဆိုတာ ဘာလဲ?

Package ဆိုတာ Code အစုအဝေး (သို့) Directory တစ်ခု ဖြစ်ပါတယ်။

  • ၎င်းထဲမှာ JavaScript ဖိုင်တွေနဲ့ အလုပ်လုပ်ပုံကို ညွှန်ပြတဲ့ package.json ဖိုင် တစ်ခု ပါဝင်ပါတယ်။
  • ဥပမာ - Date တွေကို တွက်ချက်ပေးတဲ့ Package, Form တွေကို စစ်ဆေးပေးတဲ့ Package စသဖြင့် အလုပ်တစ်ခုစီအတွက် သီးသန့်ရေးထားတဲ့ Code အစုအဝေးတွေပါ။

Module ဆိုတာ ဘာလဲ?

Node.js မှာ `require()` ဒါမှမဟုတ် `import` သုံးပြီး ခေါ်ယူနိုင်တဲ့ JavaScript ဖိုင်တိုင်းကို Module လို့ခေါ်ပါတယ်။

  • Package တစ်ခုမှာ Module တစ်ခု (သို့) အများကြီး ပါဝင်နိုင်ပါတယ်။
  • npm ကနေ ဒေါင်းလုပ်ဆွဲလိုက်တဲ့ Package တိုင်းဟာ ကိုယ့် Project အတွက် Module တွေအဖြစ် အလုပ်လုပ်ပေးပါတယ်။

npm ရဲ့ အဓိက အစိတ်အပိုင်း ၃ ခု

npm ကို အဓိက အပိုင်း ၃ ပိုင်းနဲ့ ဖွဲ့စည်းထားပါတယ် -

  1. The Website (ဝဘ်ဆိုဒ်)
  2. The Registry (ဒေတာဘေ့စ်)
  3. The CLI (Command Line Interface)

၁. The Website (npm ဝဘ်ဆိုဒ်)

npmjs.com ဆိုတာ npm ရဲ့ တရားဝင် ဝဘ်ဆိုဒ်ဖြစ်ပါတယ်။

  • အဲဒီမှာ ကိုယ်လိုအပ်တဲ့ Package တွေကို ရှာဖွေ (Search) နိုင်ပါတယ်။
  • Package တွေရဲ့ Documentation (အသုံးပြုနည်း လမ်းညွှန်) တွေကို ဖတ်ရှုနိုင်ပါတယ်။
  • Developer တွေက သူတို့ရဲ့ Profile တွေကို စီမံခန့်ခွဲနိုင်ပါတယ်။

၂. The Registry

Registry ဆိုတာ ဧရာမ Database ကြီး တစ်ခုပါ။

  • ကမ္ဘာတစ်ဝှမ်းက Developer တွေ ရေးသားထားတဲ့ Open-source JavaScript Package ပေါင်း သန်းနဲ့ချီပြီး သိမ်းဆည်းထားတဲ့ နေရာဖြစ်ပါတယ်။
  • ကျွန်တော်တို့ npm install လို့ ရိုက်လိုက်တဲ့အခါ၊ ဒီ Registry ကြီးထဲကနေ Code တွေကို လှမ်းဆွဲယူတာ ဖြစ်ပါတယ်။

၃. The CLI (Command Line Interface)

CLI ဆိုတာ Terminal (သို့) Command Prompt ကနေ စာသားတွေရိုက်ပြီး အသုံးပြုရတဲ့ စနစ်ပါ။

  • Developer အများစုက CLI ကိုပဲ အဓိက သုံးကြပါတယ်။
  • Package တွေကို သွင်းတာ (Install)၊ ဖြုတ်တာ (Uninstall)၊ အဆင့်မြှင့်တာ (Update) တွေကို CLI ကနေတစ်ဆင့် ခိုင်းစေရပါတယ်။

Node.js နဲ့ npm ဆက်စပ်မှု

npm ဟာ Node.js နဲ့အတူ အလိုအလျောက် ပါလာပါတယ်။

  • ကွန်ပျူတာမှာ Node.js ကို Install လုပ်လိုက်တာနဲ့ npm ကိုပါ တပါတည်း ရရှိမှာဖြစ်ပါတယ်။
  • သီးသန့်ကြီး npm ကို သွားရောက် Install လုပ်နေစရာ မလိုပါဘူး။
// Version စစ်ဆေးရန်
node -v
npm -v

package.json ဆိုတာ ဘာလဲ?

Project တစ်ခုရဲ့ မှတ်ပုံတင် (သို့) နှလုံးသား လို့ တင်စားလို့ရပါတယ်။

  • Project အမည်၊ Version၊ ဖန်တီးသူ စတဲ့ အချက်အလက်တွေ ပါဝင်ပါတယ်။
  • အရေးအကြီးဆုံးကတော့ Project အတွက် အသုံးပြုထားတဲ့ Package တွေရဲ့ စာရင်းနဲ့ Version တွေ ကို မှတ်သားထားပေးတဲ့ ဖိုင်ဖြစ်ပါတယ်။

package.json ရဲ့ အရေးပါပုံ

  • တခြားသူကို Project ပေးတဲ့အခါ Code တွေအကုန် (node_modules) ထည့်ပေးစရာ မလိုတော့ပါဘူး။
  • ဒီ package.json ဖိုင်လေး ပါသွားရုံနဲ့ တခြားသူက npm install ရိုက်လိုက်တာနဲ့ လိုအပ်တဲ့ Package တွေ အလိုအလျောက် ရောက်လာမှာပါ။
  • ဒါဟာ Project size ကို သေးငယ်စေပြီး မျှဝေရ လွယ်ကူစေပါတယ်။

npm init ဖြင့် Project စတင်ခြင်း

Project အသစ်တစ်ခု စတင်တိုင်း package.json ဖိုင်ကို ဖန်တီးဖို့ လိုပါတယ်။

npm init

// မေးခွန်းတွေ မဖြေချင်ဘဲ အလိုအလျောက် ဖန်တီးချင်ရင်
npm init -y

ဒီ Command ကို ရိုက်လိုက်တာနဲ့ package.json ဖိုင်လေး ထွက်ပေါ်လာပါမယ်။

npm install ဆိုတာ ဘာလဲ?

Registry ကနေ Package တွေကို ကိုယ့်စက်ထဲ ထည့်သွင်းတဲ့ Command ပါ။

npm install package_name
// အတိုကောက် အနေဖြင့်
npm i package_name
  • ဥပမာ React ကို သွင်းချင်ရင် npm install react လို့ ရိုက်ရပါတယ်။

Local vs Global Installation

Package တွေကို ထည့်သွင်းရာမှာ အဓိက နည်းလမ်း ၂ မျိုး ရှိပါတယ်။

  • Local: လက်ရှိ လုပ်နေတဲ့ Project Folder ထဲမှာပဲ အလုပ်လုပ်တာပါ။
  • Global: ကွန်ပျူတာ တစ်ခုလုံး နေရာတကာကနေ လှမ်းသုံးလို့ရအောင် ထည့်သွင်းတာပါ။

Local Installation အကြောင်း

  • ပုံမှန် npm install package_name လို့ ရိုက်ရင် Local အနေနဲ့ ဝင်ပါတယ်။
  • Package တွေဟာ Project Folder ထဲက node_modules ဆိုတဲ့ Folder ထဲကို ရောက်သွားပါတယ်။
  • အဲ့ဒီ Project အတွက်ပဲ သီးသန့် အသုံးပြုလို့ ရမှာ ဖြစ်ပါတယ်။

Global Installation အကြောင်း

  • Command line tool တွေလိုမျိုး နေရာတိုင်းက သုံးချင်တဲ့အခါ Global သွင်းရပါတယ်။
  • ဥပမာ nodemon လိုမျိုး tool တွေပါ။
// -g ကို ထည့်ရေးရပါတယ်။
npm install -g nodemon

Dependencies ဆိုတာ ဘာလဲ?

ကိုယ့် Project အလုပ်လုပ်ဖို့ မရှိမဖြစ် မှီခိုနေရတဲ့ Package တွေကို Dependencies လို့ ခေါ်ပါတယ်။

  • ဥပမာ Web Server ရေးဖို့ express ကို သုံးထားတယ်ဆိုရင် express ဟာ Dependency ပါ။
  • npm install လုပ်တဲ့အခါ package.json ထဲက "dependencies" အောက်မှာ မှတ်သားသွားပါတယ်။

DevDependencies ဆိုတာ ဘာလဲ?

Code ရေးနေစဉ် (Development) အချိန်မှာပဲ လိုအပ်ပြီး၊ တကယ့် Website လွှင့်တဲ့အချိန် (Production) မှာ မလိုအပ်တဲ့ Package တွေပါ။

  • ဥပမာ Code တွေကို အလိုအလျောက် format ချပေးတဲ့ prettier လိုမျိုး Package တွေ ဖြစ်ပါတယ်။
npm install prettier --save-dev
// (သို့) npm i prettier -D

package-lock.json ဆိုတာ ဘာလဲ?

npm install လုပ်လိုက်တာနဲ့ အလိုအလျောက် ထွက်လာတဲ့ ဖိုင်တစ်ခုပါ။

  • package.json က Version အကြမ်းဖျင်းကို မှတ်ပြီး၊ package-lock.json ကတော့ သွင်းလိုက်တဲ့ Version အတိအကျကို Lock ချပေးထားတာပါ။
  • ဒါမှသာ တခြားစက်မှာ Run တဲ့အခါ Version မကိုက်ညီတဲ့ ပြဿနာကို ရှောင်လွှဲနိုင်မှာ ဖြစ်ပါတယ်။

npm update လုပ်နည်း

မိမိသွင်းထားတဲ့ Package တွေ Version အသစ်ထွက်လာတဲ့အခါ အလွယ်တကူ Update လုပ်နိုင်ပါတယ်။

// Package တစ်ခုတည်းကို Update လုပ်ရန်
npm update package_name

// Project ထဲက Package အားလုံးကို Update လုပ်ရန်
npm update

npm uninstall လုပ်နည်း

မလိုချင်တော့တဲ့ Package တွေကို Project ထဲကနေ ပြန်ဖျက်ထုတ်ချင်တဲ့အခါ အသုံးပြုပါတယ်။

npm uninstall package_name
  • ဒီလိုရိုက်လိုက်တာနဲ့ node_modules ထဲကရော၊ package.json စာရင်းထဲကပါ အလိုအလျောက် ပျောက်သွားမှာ ဖြစ်ပါတယ်။

Semantic Versioning (SemVer)

npm မှာ Package တွေရဲ့ Version တွေကို နံပါတ် ၃ လုံး (ဥပမာ 1.4.2) နဲ့ သတ်မှတ်ပါတယ်။ ဒါကို SemVer လို့ခေါ်ပါတယ်။

  • 1 (Major): အပြောင်းအလဲကြီးမားပြီး ယခင် Code တွေနဲ့ အဆင်မပြေနိုင်တော့တဲ့ အခြေအနေ။
  • 4 (Minor): Feature အသစ်တွေ ထပ်ထည့်ထားပေမယ့် ယခင် Code တွေ ပျက်မသွားတဲ့ အခြေအနေ။
  • 2 (Patch): Bug (အမှား) လေးတွေကို ပြင်ဆင်ထားတဲ့ အခြေအနေ။

^ နှင့် ~ သင်္ကေတများ

package.json ထဲမှာ Version တွေရဲ့ အရှေ့မှာ သင်္ကေတလေးတွေ တွေ့ရတတ်ပါတယ်။

  • ^ (Caret): Minor နဲ့ Patch version အသစ်ထွက်ရင် လက်ခံမယ်။ ဥပမာ ^1.4.2 ဆိုရင် 1.5.0 ထိ တက်ခွင့်ပေးတယ်။ Major တော့ အပြောင်းမခံဘူး။
  • ~ (Tilde): Patch version အသစ်ထွက်မှသာ လက်ခံမယ်။ ဥပမာ ~1.4.2 ဆိုရင် 1.4.3 အထိပဲ တက်ခွင့်ပေးတယ်။

npm scripts ဆိုတာ ဘာလဲ?

package.json ထဲမှာ ကိုယ်ပိုင် Command လေးတွေ ဖန်တီးထားလို့ရပါတယ်။

"scripts": {
  "start": "node server.js",
  "dev": "nodemon server.js"
}
  • ဒီတော့ node server.js လို့ အရှည်ကြီး ရိုက်နေမယ့်အစား npm start ဒါမှမဟုတ် npm run dev လို့ အလွယ်တကူ ရိုက်သုံးနိုင်ပါတယ်။

နာမည်ကြီး npm packages များ

ကမ္ဘာပေါ်မှာ နေ့စဉ်သုံးစွဲနေတဲ့ နာမည်ကြီး Package တွေ အများကြီးရှိပါတယ်။

  • React / Vue / Angular (UI ဖန်တီးရန်)
  • Express (Node.js ဖြင့် Web Server ရေးရန်)
  • Lodash (JavaScript Data များကို လွယ်ကူစွာ ကိုင်တွယ်ရန်)
  • Axios (API များမှ Data လှမ်းယူရန်)

လုံခြုံရေး (npm audit)

မိမိသုံးထားတဲ့ Package တွေမှာ လုံခြုံရေး အားနည်းချက် (Vulnerabilities) ရှိမရှိ စစ်ဆေးပေးပါတယ်။

npm audit
  • အကယ်၍ အားနည်းချက် တွေ့ရှိပါက npm audit fix ဆိုတဲ့ Command ကို ရိုက်ပြီး အလိုအလျောက် ပြင်ဆင်နိုင်ပါတယ်။

npm အသုံးပြုခြင်းရဲ့ အကျိုးကျေးဇူးများ

  • အချိန်ကုန်သက်သာခြင်း: သူများရေးပြီးသား Code ကို ယူသုံးရုံပါပဲ။
  • စနစ်ကျခြင်း: Version တွေကို စနစ်တကျ ထိန်းချုပ်နိုင်တယ်။
  • လွယ်ကူခြင်း: Command တစ်ကြောင်းရိုက်ရုံနဲ့ လိုချင်တဲ့ Code တွေ ရောက်လာတယ်။
  • Community ကြီးမားခြင်း: ပြဿနာတက်ရင် ဖြေရှင်းနည်း အလွယ်တကူ ရှာတွေ့နိုင်တယ်။
